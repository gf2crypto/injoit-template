# LaTeX шаблон для журнала INJOIT
Репозиторий содержит LaTeX шаблон статьи для журнала International Journal of Open Information Technologies ([INJOIT](http://injoit.ru/)).
Шаблон основан на шаблоне [IEEEtran LaTeX](http://www.michaelshell.org/tex/ieeetran/).

Эта ветка содержит `cookiecutter`-версию ([more details](https://cookiecutter.readthedocs.io/en/latest/)) LaTeX-шаблона.

__Как использовать шаблон__:
1. Установите `python` и `git`;
2. Установите приложение `cookiecutter`, выполнив в командной строке следующую команду:
   ```bash
   pip install cookiecutter
   ```
4. Для инициализации нового проекта выполните в командной строке:
   ```bash
   cookiecutter https://github.com/gf2crypto/injoit-template.git -c cc
   ```
5. Параметры:
     - **project_name**, имя проекта (поддерживается только латиница);
     - **create_gitignore**, требуется ли создать `.gitignore` файл для использования системы контроля версий `git`: 'yes' или 'no'.
6. Для компиляции документа можно использовать команду `make` (`make.bat` под `Windows`).
7. Запустите `make help` (`make.bat help` для `Windows`), чтобы увидеть другие возможные опции компиляции. Рекомендуется обратить внимание на опцию `watch`, которая обеспечивает непрерывную компиляцию проекта и позволяет видеть изменения в `pdf` практически в режиме реального времени. __Обратите внимание__, что `make`-файлы используют `latexmk` для компиляции документа. Возможно придётся установить этот пакет. Рекомендуется устанавливать __полную__ версию `LaTeX`, чтобы сэкономить время на отладке и доустановке пакетов!
6. Также возможна компиляция вручную. __ВНИМАНИЕ__! Шаблон не поддерживает компиляцию с помощью `pdflatex`! Используйте `xelatex` (он входит в состав большинства LaTeX дистрибутивов).
Для компиляции вручную можете выполнить следующую последовательность команд:
```bash
xelatex project_name
biber project_name
xelatex project_name
biber project_name
xelatex project_name
```

## Использование шаблона
 * Параметры задаются в файле `config.tex`.
 * Помещайте рисунки в папку `images` и они будут автоматически "подхватываться" шаблоном
 * Библиографию помещайте в файл `%_ru.bib`.
 * По требованиям журнала нужно сформировать библиографию на английском языке для этого англоязычные источники остаются без изменений, а русскоязычные пишутся транслитом, а также переводятся на английский. Английскую версию библиографии помещайте в файл `%_en.bib`. Не забудьте переименовать ключи библиографических записей в английской версии, чтобы не было дубликатов (см. пример в файле `%_en.bib`).
 * Не забудьте в финальном варианте указать выходные данные статьи и номер первой страницы (см. конец файла `config.tex`)
